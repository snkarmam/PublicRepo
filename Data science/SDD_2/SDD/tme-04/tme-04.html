Complments sur le perceptron

Afin d'amliorer notre classifieur perceptron, nous allons considrer une version modifie de la classe ClassifierPerceptronBiais, que nous appellerons ClassifierPerceptronBiais pour construire un perceptron en utilisant un biais qui facilite la convergence du modle: pour dterminer si une mise jour des poids doit tre faite, le critre de mauvaise classification est remplac par $$ f(\mathbf x_i) y_i < 1 $$ o $f(\mathbf x_i)$ est le score obtenu pour $x_i$.

Ide: on veut que $f(\mathbf x_i)$ et $y_i$ soient du mme signe ET que $f(\mathbf x_i)$ soit suffisamment grand (en valeur absolue).

C'est une version dite stabilise du perceptron o les points ne sont considrs bien classs que lorsque $f(\mathbf x_i) y_i \geq 1$. Dans le cas contraire, on met jour les poids.

La mise jour des poids tient alors compte de ce biais: $$ w = w +\epsilon (y_i- f(\mathbf x_i))x_i.$$

Nous utilisons ici une version stabilise du perceptron o les points ne sont bien class que lorsque $f(\mathbf x_i) y_i>1$

Validation Croise

Dans le but d'valuer un classifier, nous avons vu dans le TME 3 qu'il tait important de possder un ensemble de donnes de test, diffrent de l'ensemble d'apprentissage.

Nous allons voir maintenant une mthode encore plus efficace pour bien valuer un algorithme.

La procdure de la validation croise (ou cross validation) est au centre de la plupart des applications de machine learning: il est temps pour nous de travailler sur une implmentation.

L'ide est de concevoir la fonction suivante:

Xapp,Yapp,Xtest,Ytest = crossval(X, Y, n_iterations, iteration)
  • X, Y sont les donnes du dataset mlanges alatoirement (on ne mlange pas physiquement les donnes, mais seulement leurs indices)
  • n_iterations est le nombre d'ensembles de test au total.
  • iteration est l'itration concerne: on ne renvoit pas les mmes donnes en fonction des itrations.

Aprs sparation des indices en n_iterations groupes, isoler 1 groupe pour le test et les autres pour l'apprentissage.

Remarques:

  • on fait l'hypothse que le dataset (X, Y) garde toujours le mme ordre entre 2 appels de crossval successif avec des valeurs d'itration diffrentes.
  • cette fonction ne doit pas mlanger les donnes du dataset fourni (qui doit tre mlang au pralable), elle sert juste extraire de ce dataset 2 sous-datasets: un dataset d'apprentissage (Xapp, Yapp) et un dataset de test (Xtest, Ytest).
  • elle extrait les datasets demands comme suit:
    • le dataset de test pour l'itration $i$ contient les exemples du dataset $X$ dont les indices vont de $i\frac{len(X)}{n}$ $(i+1)\frac{len(X)}{n}-1$ (avec $n$ le nombre d'itrations n_iterations fix).
    • le dataset d'apprentissage pour l'itration $i$ contient les exemples du dataset $X$ contient les exemples qui ne sont pas dans le dataset de test.

Note Pour mlanger les donnes au dpart, avant le premier appel de crossval, vous pouvez utiliser les commandes suivantes:

index = np.random.permutation(len(X)) # mlange des index
Xm = X[index]
Ym = Y[index]

# check: malgr le mlange, les donnes doivent tre les mmes

plt.scatter(Xm[Ym==-1,0], Xm[Ym==-1,1], c='r')

Plus loin avec le multi-classes

Le perceptron est un algorithme fondamentalement bi-classes: il est mme li la manire dont sont encodes les tiquettes $\{+1,-1\}$. La plupart des problmes rels sont pourtant multi-classes (e.g. USPS). Si cet algorithme a pu rester aussi classique, c'est qu'il existe un moyen d'tendre un classifieur binaire en une extension multi-classes simple et efficace: le un-contre-tous (ou One-Against-All en anglais):

(si l'image ci-dessous ne s'affiche pas, aller la voir dans le rpertoire ressources)

Dans l'approche un-contre-tous l'ide est d'apprendre autant de classifieurs binaires que de classes.

Dans l'exemple ci-dessus, cela donne:

  1. Traitement de la classe 0
  2. redfinition de: $$Y_{tmp} = \left\{ \begin{array}{ll} 1 & \text{ si } Y = 0\\ -1 & \text{ si } Y \neq 0\\ \end{array} \right. $$
  3. apprentissage de $f_0$ sur $X,Y_{tmp}$
  4. Traitement de la classe 1
  5. redfinition de: $$Y_{tmp} = \left\{ \begin{array}{ll} 1 & \text{ si } Y = 1\\ -1 & \text{ si } Y \neq 1\\ \end{array}\right. $$
  6. apprentissage de $f_1$ sur $X,Y_{tmp}$
  7. etc...

En infrence, lorsqu'un nouveau point $\mathbf x$ est prsent il faut:

  1. calculer $score = [f_0(\mathbf x), f_1(\mathbf x), \ldots f_C(\mathbf x)]$
  2. prdire $\text{argmax}_c (score)$

Pour implmenter cela, on ne va pas dfinir une version multi-classe pour chaque classifieur binaire que l'on a crit, cela ne serait pas de la programmation efficace... On va dfinir un nouveau classifieur gnrique multi-classes qui utilisera un classifieur binaire (en POO, on va se baser sur le pattern composite) en respectant le schma suivant:

  1. ClassifierMultiOAA prend un classifieur binaire en argument lors de la cration. Il initialise une liste vide de classifieurs.

  2. Dans la mthode train, penser cloner le classifeur de rfrence nCl fois (autant de classifieurs que de classes dans la base d'apprentissage). Il faut utiliser la mthode deepcopy du module standard python copy

  3. Dans un boucle for, redfinir les tiquettes $ytmp$ et apprendre des classifieurs binaires

  4. Dans score faire appel aux mthodes score des classifieurs binaires et stocker les rsultats

  5. Dans predict renvoyer l'argmax des scores.
    Note: en faisant cette opration, on fait l'hypothse que les tiquettes sont dfinies sur $[0,nCl]$. Pour lever cette hypothse, il faudrait stocker les classes et convertir le rsultat de argmax.

Important: La fonction accuracy de la classe mre doit tre compatible avec les dfinitions prcdentes... Par exemple:

def accuracy(self, desc_set, label_set):
        yhat = np.array([self.predict(x) for x in desc_set])
        return np.where(label_set == yhat, 1., 0.).mean()

Attention: on fait ici l'hypothse que l'on ne considre que des classes numrotes de $0$ $9$, comme celles dans USPS. Ainsi, le premier classifieur reconnat la classe $0$ contre toutes les autres, le deuxime classifieur reconnat la classe $1$ contre les autres, etc. Pour une solution plus gnrale, la fonction predict ne devra pas rendre l'indice du maximum (argmax) mais la classe qui correspond ce maximum.
